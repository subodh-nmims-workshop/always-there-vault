## 🏗️ 2. Dual-Mode Architecture

The protocol uniqueness lies in its dual-mode capabilities, allowing users to choose their optimal balance between convenience and self-sovereignty.

### 🏢 Centralized Mode (Managed Service)
*   **Architecture:** NestJS + MongoDB + AWS S3 / IPFS Backup.
*   **Pricing:** Starter ($9.99), Professional ($29.99), Enterprise ($99.99)
*   **Target:** Non-technical users, mobile-first users, businesses.
*   **Pros:** Fast setup, no wallet needed, 24/7 support, account recovery.

### ⛓️ Decentralized Mode (Self-Sovereign)
*   **Architecture:** Smart Contracts + IPFS + Arweave.
*   **Pricing:** Freedom ($19.99), Sovereign ($49.99), Immortal ($149.99 / $1,499 one-off) + user pays gas fees.
*   **Target:** Privacy advocates, crypto natives, users needing censorship resistance.
*   **Pros:** Zero-knowledge privacy, true ownership, censorship resistance.

**Seamless Mode Switching:** A core feature allows users to map and migrate their assets between modes at any time.

## 🚀 3. Getting Started & Setup Guide

### The "One Command to Rule Them All" Start
From the project root, simply run:
```bash
./start.sh
```
This automated script will:
1. Start the Hardhat local blockchain (Port 8545)
2. Deploy the smart contracts
3. Start the NestJS Backend API (Port 3001 or 3100)
4. Start the Next.js Frontend Web (Port 7000)

### Graceful Terminations
To safely exit and stop all services:
```bash
./stop.sh
```

### Environment Setup Requirements
*   **Backend (`backend/.env`):** `DATABASE_URL`, `JWT_SECRET`, `STRIPE_SECRET_KEY`, `STRIPE_PUBLISHABLE_KEY`.
*   **Frontend (`frontend/web/.env.local`):** `NEXT_PUBLIC_API_URL`, `NEXT_PUBLIC_CONTRACT_ADDRESS`, `NEXT_PUBLIC_STRIPE_PUBLIC_KEY`.
